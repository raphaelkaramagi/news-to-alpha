# Evaluation results

**Reproduce:**

```bash
python scripts/build_eval_dataset.py
python scripts/build_ensemble.py --conditional
python scripts/evaluate_predictions.py --horizon 1
cat data/processed/evaluation_summary.txt
```

## Volatility / expected move

The volatility model is separate from direction. It predicts **how large** the next session move will be, not which way price goes.

### Target

For each (ticker, date **T**), the label is the **absolute** close-to-close return to the next session:

`|return| = |close(T+1) / close(T) − 1| × 100` (percent)

The model outputs `expected_move_pct` — a point estimate of that |return| in percent. The dashboard shows it as **±X%** around the last close (symmetric band: `close × (1 ± X/100)`).

**Model:** `HistGradientBoostingRegressor` on 12 price features (realized vol, ATR, Bollinger width, volume z-score, overnight gap, VIX level/change, etc.). Same chronological train/val/test split as the LSTM.

### Metrics (test split, n = 2,220)

| Metric | What it measures | Value |
|--------|------------------|-------|
| **High-move AUC** | Can the model rank days with above-median \|return\| above days with below-median \|return\|? (1 = perfect ranking, 0.5 = random) | **0.649** |
| **MAE** | Average absolute error between predicted and realized \|return\|, in percentage points | **1.22%** |
| **Median \|return\|** | Typical realized move size on the test set (reference scale for MAE) | **1.21%** |

**How to read this:** Direction models sit at AUC ≈ 0.50; volatility at **0.649** means next-day move *size* is genuinely more predictable than direction. MAE ≈ 1.22% on a median move of ≈ 1.21% means the band is roughly calibrated in magnitude — not exact per day, but useful as an expected range.

Printed at the end of `evaluate_predictions.py` under **Volatility / Expected-Move Diagnostics**.

### Not in the direction ensemble

`expected_move_pct` was tested as an extra ensemble feature. Direction AUC did **not** improve (`news_scored` ensemble AUC **0.485** with vol vs **0.492** without). The ensemble stays 13-feature direction-only; volatility is a **standalone** dashboard output.

## Does direction work better on quiet days?

Rows are split by predicted vol regime: **`high_vol`** = `expected_move_pct` ≥ train-set median (~1.0%); **`low_vol`** = below median. This asks whether direction is easier when the model expects a small move.

| Subset | n | Ensemble AUC | LSTM AUC | Notes |
|--------|---|--------------|----------|-------|
| `low_vol` | 339 | 0.507 | 0.548 | LSTM slightly higher, but **small n** — not reliable |
| `high_vol` | 1,881 | 0.508 | 0.495 | No edge |

Neither regime gives ensemble direction AUC meaningfully above 0.50. The low-vol LSTM bump is not large enough or stable enough to change the product story. Full tables: `evaluation_summary.txt` (subsets `low_vol`, `high_vol`).

## Drift base rates

Accuracy from always predicting UP — reflects **market drift**, not model skill:

| Horizon | Always-UP accuracy |
|---------|-------------------|
| 1 day | ~52.5% |
| 5 day | ~54.3% |
| 20 day | ~58% |

Compare these to **AUC**, not raw accuracy, when judging predictive skill.
